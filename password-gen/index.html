자주 묻는 질문

이 비밀번호 생성기는 안전한가요?

네. 이 랜덤 패스워드 생성기는 crypto.getRandomValues()를 사용해 암호학적으로 안전한 난수를 생성합니다. Math.random()과 달리 예측이 불가능하며, 모든 처리는 브라우저 안에서만 이루어집니다. 생성된 비밀번호는 서버로 전송되지 않습니다.

생성된 비밀번호는 저장되나요?

아니요. 이 도구는 완전히 브라우저에서만 동작하는 클라이언트 완결 도구입니다. 생성된 비밀번호는 서버로 전송되거나 외부에 기록되지 않습니다. 새로고침하면 화면에서도 사라집니다.

비밀번호 길이는 얼마가 적당한가요?

최소 12자 이상을 권장하며, 중요한 계정(이메일·금융·업무)에는 16자 이상이 바람직합니다. 길이가 길수록 무차별 대입 공격에 필요한 시간이 기하급수적으로 늘어나기 때문에 강력한 비밀번호 만들기의 첫 번째 조건입니다.

특수문자를 포함해야 하는 이유는 무엇인가요?

특수문자를 포함하면 패스워드의 엔트로피(정보량)가 크게 높아집니다. 예를 들어 소문자만 26가지인 데 반해, 대·소문자·숫자·특수문자를 모두 쓰면 약 94가지 문자를 사용할 수 있어 같은 길이라도 무차별 대입 저항성이 비약적으로 향상됩니다.

생성한 비밀번호를 어디에 저장하면 되나요?

패스워드 매니저(예: Bitwarden, 1Password, KeePass 등) 사용을 권장합니다. 강력한 비밀번호를 계정마다 다르게 사용하면서도 외울 필요가 없고, 유출 알림 기능도 제공합니다. 메모장이나 엑셀에 평문 저장하는 방식은 피하세요.
